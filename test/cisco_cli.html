コマンドの「意味」が分かる！
Cisco CLI 設定スライドガイド

「今、何のためにこのコマンドを入力しているんだろう？」という疑問を徹底解決。 設定の理由と、もし設定しなかった場合の現場トラブルをスライド形式で楽しく学びます。

作成・指導

サークル長: 永井 秀汰

「ただコマンドを打つだけ」になっていませんか？

暗記作業の限界

「黒い画面に指示書通りのコマンドを打つ」だけだと、トポロジが少し変わったり、配線ミスをした瞬間に、何が原因か全く分からなくなってしまいます。

目的が分かると「応用」ができる！

コマンド1行ずつの「設定する理由」や「設定しないとこうなる」を理解すれば、実習やトラブルシューティング（障害復旧）の際に自分で考えて解決できるようになります！

※ 次のスライドから、具体的なCiscoコマンドの「目的」を学んでいきましょう！

ルータ基本設定 ※最初に超推奨

なぜ最初に no ip domain-lookup を設定するの？

対象コマンド：

Router(config)# no ip domain-lookup

設定しないとどうなる？（恐怖のタイポフリーズ）

例えば conf t と打つべきところを、焦って conft とタイポすると、ルータは「conftという名前のDNSサーバー」を世界中へ探しに行ってしまい、数分間入力を完全に受け付けないフリーズ状態になります。

設定する目的・メリット

このコマンドを入力しておくことで、「タイポをドメイン名（URL）と解釈してDNS検索を行う機能」をオフにできます。打ち間違えてもフリーズせず、一瞬で「そんなコマンドありません」と返してくれるので実習が劇的に快適になります。

ルータ基本設定 ※忘れると泣くやつ

なぜ苦労して設定した後に copy run sta を打つのか？

対象コマンド：

Router# copy running-config startup-config (省略形: cop run sta)

設定しないとどうなる？（再起動で全消失）

Ciscoルータの設定変更は、一時的なメモリ（RAM）上で動いています。保存せずに放置した状態で、 Packet Tracer上で誤って再起動させたり、PCが急に落ちたりすると、それまで頑張って打ったすべての設定が跡形もなく消え去ります。

設定する目的・メリット

RAMで動いている現在進行形の設定（running-config）を、ルータ内部の不揮発性メモリ（NVRAM）にある起動用設定（startup-config）に書き込みます。これにより、電源を落としても設定がしっかりと保持されます！

アドレッシング 地味に超大事

なぜインタフェース設定で no shutdown と打つの？

対象コマンド：

Router(config-if)# no shutdown (省略形: no sh)

設定しないとどうなる？（ポートは永眠状態）

Ciscoルータのイーサネットなどのポート（インタフェース）は、セキュリティと節電のため、工場出荷状態でデフォルト「shutdown（無効）」になっています。どれだけ完璧にIPアドレスを設定しても、ケーブルを差しても、ランプは赤色のままで通信ができません。

設定する目的・メリット

「shutdown を no（打ち消す）」、つまり「停止状態を解除する」という意味になります。このコマンドを打った瞬間に、ポートのリンクアップランプが緑色に変化し、初めてそのポートで通信のやりとりができる電気的な準備が整います。

アドレッシング ネットワークの基本

IPアドレスを設定することの本当の意味

対象コマンド（IPv4 / IPv6）：

Router(config-if)# ip address 192.168.0.1 255.255.255.0 Router(config-if)# ipv6 address 2001:db8:1::1/64

なぜIPアドレスと「マスク」をセットで打つの？

IPアドレスだけでは、そのルータが「何市（ネットワーク）」の「何丁目何番地（ホスト）」なのかが分かりません。サブネットマスク（プレフィックス）を併記することで、ルータは**「自分と同じグループ（LAN）」と「違うグループ（WAN）」の境界線**を判断しています。

なぜ IPv6 では ipv6 unicast-routing が必要なの？

Ciscoルータはデフォルトで「IPv4」のパケットしかルーティング（転送）してくれません。これを設定しないと、いくら各ポートに美しいIPv6アドレスを設定しても、ルータはIPv6パケットを受け取った瞬間にルーティングせずに捨ててしまいます。

RIPルーティング 動的ルートの極意

なぜ network や version 2、no auto-summary がいるの？

対象コマンド：

Router(config-router)# network 192.168.0.0 Router(config-router)# version 2 / no auto-summary

network は何をお知らせしている？

ルータが持っている「私は192.168.0.0のネットワークを知ってますよ！」という情報を、隣接する他のルータに「おすそ分け（宣伝）」するための設定です。これをお互いにやることで、直接繋がっていない遠くの道（ルート）をルータ同士が学習します。

version 2 と no auto-summary を忘れると？

デフォルト(V1)だと、勝手にアドレスを「クラスA/B/C」の境界にざっくりと省略（集約）して他人に伝えてしまいます。せっかく細かくサブネットマスクで分けたのに、他のルータに正しく伝わらず、パケットが行き先を見失って迷子になります。

DHCPv4設定 自動化の恩恵

なぜ excluded-address（除外）が必要なの？

対象コマンド：

Router(config)# ip dhcp excluded-address 192.168.0.1 192.168.0.10

DHCPを設定する理由

PCが何百台もある会社で、全PCに手動でアドレスやDNSのIPを叩き込むのは不可能です。ルータにDHCP設定を施すことで、LANにPCを繋いだ瞬間に、自動的かつ重複なく、必要なIP設定情報を自動レンタルしてくれます。

除外(excluded)を忘れるとIPアドレスが競合する！

デフォルトのままだと、ルータは「自分のポート(例:192.168.0.1)」と同じアドレスも、一般のPCに勝手に配布してしまいます。これによりIPの重複（競合）が発生し、PC側でもルータ側でも通信ができなくなる重大なトラブルに発展します。
