ルータ基本設定 重要度：★★★

なぜ最初に「no ip domain-lookup」を打つの？

実行コマンド（グローバルコンフィグモード）：

Router(config)# no ip domain-lookup

🔴 設定しないと（タイポフリーズ）

コマンドを焦ってスペルミス（例: conft）した瞬間、ルータは「conftという名前のサーバーがネット上にあるかも！」と解釈し、DNS探索を勝手に開始します。この間、約2分〜3分間キー入力を一切受け付けず、画面が完全に固まります。

🟢 設定する本当の理由

「タイポ（打ち間違い）をインターネットのドメイン名と誤解して検索しに行く機能」を停止させます。設定後はタイポしてもフリーズせず、一瞬で「エラー」と返して次の入力を進められるようになります。

ルータ基本設定 忘れがち注意

なぜ苦労して設定した後に「保存コマンド」を打つの？

実行コマンド（特権EXECモード）：

Router# copy running-config startup-config (省略形: cop run sta)

🔴 設定しないと（再起動で全消え）

ルータを動かしながら打ち込んだ設定（running-config）は、電源を切ると消えてしまう揮発性メモリ（RAM）上で動作しています。保存を忘れてルータの電源が落ちたり、Packet Tracer上で再起動をかけると、全てが消え去りやり直しになります。

🟢 設定する本当の理由

動いている設定データを、電源を切っても消えない不揮発性メモリ（NVRAM）の初期起動ファイル（startup-config）へ書き移して保存します。これでルータをいつ再起動しても設定がしっかり保持されます！

インタフェース設定 リンクアップの基本

なぜインタフェース設定で「no shutdown」を打つの？

実行コマンド（インタフェースコンフィグモード）：

Router(config-if)# no shutdown (省略形: no sh)

🔴 設定しないと（通信はずっと不通）

Ciscoルータの各物理ポートは、購入時の初期状態で「セキュリティと節電のため、無効（shutdown状態）」に設定されています。IPアドレスを完璧に入力し、ケーブルを正しく差しても、ポートを起こさない限り通信の電気信号は一切通りません。

🟢 設定する本当の理由

「shutdown（止める）を no（打ち消す）」、つまり「停止を解除して稼働させる」という命令です。このコマンドを打つことでポートが起動状態になり、Packet Tracer上のリンクランプが緑色に変化してデータが通るようになります。

アドレッシング IPv4 / IPv6

なぜIPアドレスとマスク（プレフィックス）をセットで打つの？

実行コマンド例（IPv4 / IPv6）：

Router(config-if)# ip address 192.168.0.1 255.255.255.0 Router(config-if)# ipv6 address 2001:db8:1::1/64

💡 サブネットマスクの役割

IPアドレスだけでは、ルータは「どこまでが同じグループ（同じネットワーク）で、どこからが外の世界なのか」の判別ができません。サブネットマスク（IPv6のプレフィックス長 `/64` 等）を指定することで、グループの境界線を示してあげています。

💡 ipv6 unicast-routing の存在理由

Ciscoルータはデフォルト状態では「IPv4の郵便屋さん」としてしか働きません。グローバルコンフィグでこの一文を打たないと、いくらポートに綺麗なIPv6アドレスを割り振っても、ルータは「IPv6パケットの転送業務」をサボり、全てその場で捨ててしまいます。

動的ルーティング RIPv2の設定

なぜ「version 2」と「no auto-summary」が必要なの？

実行コマンド（ルータRIPモード）：

Router(config-router)# version 2 Router(config-router)# no auto-summary

🔴 指定を忘れてしまうと（通信不良）

デフォルト（バージョン1）のままだと、ルータが周辺ネットワークに自分のルートを広告する際、サブネットマスクの情報を切り捨てて「クラスフル」な境界（大雑把なネットワーク範囲）で勝手に集約して伝えてしまいます。そのせせいでパケットが間違ったルータへ向かい、途中で消失します。

🟢 設定する本当の理由

version 2でクラスレスなやり取りを有効化し、no auto-summaryで余計なルート集約をオフにします。これで自分が細かく設計したそのままの正確なサブネット情報が隣のルータに正しく伝わります。

DHCP設定 IPアドレスの除外

なぜ「excluded-address（除外）」を設定するの？

実行コマンド（グローバルコンフィグモード）：

Router(config)# ip dhcp excluded-address 192.168.0.1 192.168.0.10

🔴 指定しないと（IPアドレスの重複）

ルータ自身に設定したアドレス（例: 192.168.0.1）や固定設定したいサーバーのアドレスを除外しないままだと、ルータは「このアドレスも空いてるな」と勘違いし、LAN内の一般PCに自動で割り当てて配ってしまいます。この結果、IPアドレスがぶつかり（競合）通信が破壊されます。

🟢 設定する本当の理由

ルータ自身のポートやデフォルトゲートウェイ、固定IPが必要な特殊なサーバー等にすでに使われている大切な範囲のIPアドレスを「DHCPの自動レンタル対象から強制的に除外」して保護するためにあらかじめ設定します。
